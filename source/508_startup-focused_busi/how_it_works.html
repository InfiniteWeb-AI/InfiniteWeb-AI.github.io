How It Works

Learn how to use filters, comparison tables, bookmarks, and the Startup Stack Wizard to build and refine your startup software stack.

Section: Filters

Using filters to narrow down tools

Filters on tools_category.html help you move from a long list of options to a focused shortlist that matches your budget, team size, and requirements.

  1. Start from any category link (for example CRM, Project Management, or Accounting & Finance) in the main navigation or from an article. This opens tools_category.html with a categoryId URL parameter such as:

    tools_category.html?categoryId=crm

    The categoryId corresponds to a ToolCategory.id value like crm, project_management, or customer_support.

  2. On the category page, you can filter tools by pricing and quality:

    • Price per user / per month – based on fields like starting_price_per_user_month and billing_model (for example, per_user_per_month, free).
    • Flat monthly price – uses starting_price_per_month to find tools under a specific budget.
    • Free forever – uses has_free_forever_plan so you can see tools that stay free beyond a trial.
    • Rating threshold – filters by average_rating (from 0–5) so you can focus on higher-rated tools.

    Each time you adjust these filters, the listing updates to show only the Tool entries that match.

  3. Many filters help you match tools to your team structure:

    • Company size – based on supported_company_sizes (for example solo, one_to_ten, eleven_to_fifty).
    • Team type – uses team_type values like remote_teams, in_office_teams, or freelancers.

    Combining these with price filters quickly surfaces tools tuned to your startup phase and working style.

  4. Use feature and integration filters when you know exactly what capabilities you need:

    • Feature tags – based on features and specific boolean flags such as includes_time_tracking or includes_live_chat.
    • Integrations – match tools that connect to your stack using values in integrations such as gmail, google_workspace, or slack.

    These filters ensure the tools you shortlist fit smoothly into your existing workflows.

  5. To see filters in context, you can open the CRM category directly. The URL encodes the category selection via categoryId=crm:

    Try filtering CRM tools

    Once on tools_category.html, experiment with price, rating, team size, feature, and integration filters. When you return to this page, you can add or change the ?section= parameter (for example, ?section=comparisons) to jump straight to another explanation below.

Building and reading comparison tables

Comparison tables let you see multiple tools side by side on compare.html, using the currently active ComparisonSet.

  1. Throughout the site you will see actions like “Add to comparison” on:

    • Tool listing cards on tools_category.html
    • Individual tool review pages (for example product.html?slug=...)
    • Wizard results pages after running the Startup Stack Wizard
    • Tool tables or callouts inside comparison articles

    When you add a tool, its Tool.id is stored in the active ComparisonSet.tool_ids array.

  2. Once you have selected at least two tools, you can open the comparison page. It reads the active ComparisonSet (for example, from local storage or your session) and renders one column per Tool.

    View a comparison table

    If you have not added any tools yet, this page will either be empty or prompt you to go back and choose tools to compare.

  3. On compare.html, each tool becomes a column in a grid. Typical rows include:

    • Basic info – tool name, category (via Tool.category_id), and short summary.
    • Pricing – based on billing_model, starting_price_per_user_month, and starting_price_per_month.
    • Ratings – uses average_rating and optionally rating_count.
    • Key features – lists selected items from features (for example, time_tracking, live_chat).
    • Integrations – highlights major values from integrations like slack or google_workspace.

    This layout makes it easy to scan differences and choose the tool that best matches your priorities.

  4. Most comparison UIs let you add or remove tools directly from the table. When you do this:

    • Removing a tool clears its Tool.id from ComparisonSet.tool_ids.
    • Adding a new tool appends its ID to the same list.

    The table redraws to match the current ComparisonSet, so any page using it stays in sync.

Saving and bookmarking tools

Bookmarks let you maintain a shortlist of tools as you browse listings, read reviews, and explore wizard results.

  1. A small bookmark or “Save” control appears next to most tool references, including:

    • Tool cards on tools_category.html
    • Individual tool review pages
    • Wizard result lists after you run the Startup Stack Wizard
    • Tool sections inside articles and comparison posts

    Clicking this control creates a SavedTool entry tied to the underlying Tool.id, the time you saved it (saved_at), and where you saved it from (source_context such as tool_listing or wizard_results).

  2. When you save a tool:

    • A new SavedTool record is added for that Tool.id.
    • Any page that shows your shortlist (for example, a dedicated “Saved Tools” view or a mini sidebar) reads from the same saved records.
    • Unsaving removes or marks that SavedTool entry as inactive, and the change is reflected everywhere your saved tools are displayed.

    You do not need to manage this state manually; the same underlying saved list is reused across listings, product pages, comparisons, and wizard results.

  3. Your saved tools are accessible from dedicated parts of the site. For example, there is a “Saved Tools” option in the main navigation at the top of every page.

    Because all pages read from the same SavedTool records, updating your shortlist in one place automatically updates it everywhere else.

  4. Bookmarks are designed to work together with comparison tables and wizard results:

    • From a comparison table, you can bookmark tools you want to revisit later.
    • From wizard recommendations, you can save the tools that look most promising.
    • From articles, you can bookmark tools as you read through curated lists and editor picks.

    Over time, this gives you a curated shortlist of tools that can be compared, reviewed, and refined before you make purchase decisions.

Startup Stack Wizard

Use a short, guided flow to generate a recommended stack of tools for your startup, grouped by category and aligned with your budget.

  1. 1. Describe your company profile

    The wizard collects key inputs such as your company_size_segment (for example one_to_ten or eleven_to_fifty) and industry (such as saas, ecommerce, or agency).

  2. 2. Set your monthly budget

    You specify a monthly_budget_limit for your whole stack. The wizard uses pricing data from each Tool (for example starting_price_per_user_month and starting_price_per_month) to recommend combinations that stay within that limit.

  3. 3. Review category-based recommendations

    The wizard creates a StartupStackRecommendation with multiple StartupStackRecommendationItem records. Each item links a Tool to a category_id (for example crm, project_management, or marketing_automation) and assigns a rank to show the suggested order within that category.

Launch the Startup Stack Wizard

After running the wizard, you can open category pages, add tools to comparisons, and bookmark your favorites directly from the results.
